# Changelog

All notable changes to this repository are documented here. This file follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/), and the project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

Per-package changelogs are generated by [Changesets](https://github.com/changesets/changesets) on release and live alongside each package at `packages/loanscope/<name>/CHANGELOG.md`. This root changelog captures repo-wide milestones that do not map cleanly to a single package.

## [Unreleased]

### Added

- Publication scaffolding for the `@loanscope/*` scope: per-package `description`, `keywords`, `README.md`, and `LICENSE`; root `CHANGELOG.md`; [`@arethetypeswrong/cli`](https://github.com/arethetypeswrong/arethetypeswrong.github.io) wired as `pnpm attw`; [Changesets](https://github.com/changesets/changesets) initialized with fixed versioning across all 13 loanscope packages.
- Release workflow at `.github/workflows/release.yml` targeting public npm (`registry.npmjs.org`), gated behind manual dispatch with a dry-run default until an `NPM_TOKEN` is provisioned.
- `@loanscope/program-rules` package consolidating the `toProgramRules` / `mergeRules` / `resolveVariant` / `findApplicableTier` primitives previously duplicated across `@loanscope/engine` and `@loanscope/products`.

### Changed

- Repository hygiene refactor: four large monolith files decomposed into single-responsibility modules, five missing barrel exports added with 42 consumer imports rewired, TypeScript strictness tightened (`noImplicitOverride`, `noFallthroughCasesInSwitch`, `forceConsistentCasingInFileNames`), every discriminated-union `switch` now terminates with an `assertNever` default, and two long-standing double-assertion casts replaced with proper type guards and a `withTx` boundary helper.
- Documentation consolidation: top-level documentation rewritten to reflect post-refactor state with a full 13-package layout table and release-ready package summary.
- Top-level `README.md` rewrite: accurate one-paragraph pitch, explicit architecture and scope sections, runnable CLI examples, linked workspace layout.

### Fixed

- `@workspace/ui` calendar component: widened `CalendarChevron` `orientation` union to `"left" | "right" | "up" | "down"` to match `react-day-picker`'s current `Chevron` slot contract; added `ChevronUpIcon` rendering for the new `"up"` case.

## [0.1.0] — Initial release

Initial public release of the `@loanscope/*` package family. At this cut:

- 13 workspace packages (`domain`, `graph`, `math`, `program-rules`, `calculations`, `config`, `products`, `lenders`, `engine`, `compare`, `sim`, `db`, `cli`) with deliberate layering and zero circular dependencies.
- Full Agency / Government / Portfolio product coverage: Fannie Mae, Freddie Mac, FHA (including Streamline), VA (including first-class `IrrrlRefi` enum), USDA (including Streamline), UWM Jumbo Pink, Prime Jumbo, Prime Jumbo Max.
- Deterministic decimal math throughout (`decimal.js`), branded primitive types for every domain quantity, strict TypeScript configuration (`strict`, `noUncheckedIndexedAccess`, `exactOptionalPropertyTypes`, `noImplicitReturns`).
- Typed computational DAG with provenance tracking, scoped evaluation, `BlockedNode` surfacing for missing inputs, and 18 first-class eligibility checks with margin computation for goal-seek.
- SQLite + Drizzle persistence adapter covering saved scenarios with versioning and restore, saved comparisons, saved simulations, custom product sets, catalog import pipeline with content hashing, and reproducible audit sessions.
- Commander-based CLI with 40+ commands spanning quote / evaluate / compare / goalseek / simulate / diff / batch / export-scenario plus the full `db` subcommand tree.
- 690+ tests across 26 test tasks, zero skipped.
